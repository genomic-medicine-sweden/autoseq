## Alignment

### Adapter Trimming

#### FastP

FastP is a versatile tool designed for trimming adapters and filtering low-quality reads from sequencing data. It ensures that only high-quality reads are retained for downstream analysis, improving the reliability of results. By default, trimmed FASTQ files are not included in the output directory.

### Mapping

#### BWAMEM2

BWAMEM2 is an optimized alignment tool for mapping sequencing reads to a reference genome. Known for its speed and memory efficiency, it is particularly suited for large-scale genomic datasets. Alignment files generated by BWAMEM2 are considered intermediate and are not published by default.

### Duplicate Marking

#### Picard MarkDuplicates

Picard MarkDuplicates is utilized to identify and label PCR duplicates that may arise during the library amplification process. Addressing these duplicates is crucial as they can artificially inflate coverage metrics, potentially leading to inaccurate confidence in variant calls. This tool processes alignment files generated by Bwa-mem2 and outputs the results in BAM format by default.

<details markdown="1">
<summary>Output files for Non UMI samples:</summary>

- `{outdir}/alignment/`
  - `*_dedup.bam`: alignment file in bam file format.
  - `*_dedup.bai`: index file for corresponding bam file.

</details>

### UMI Processing

#### Fgbio

Fgbio is a toolkit designed for handling sequencing data with Unique Molecular Identifiers (UMIs). It facilitates the creation of consensus reads, reducing sequencing errors and improving the accuracy of variant detection in UMI-tagged datasets.

<details markdown="1">
<summary>Output files for UMI samples:</summary>

- `{outdir}/alignment/`
  - `*_consensus_mapped.bam`: alignment file in bam file format.
  - `*_consensus_mapped.bai`: index file for corresponding bam file.

</details>

### Germline SNV and small Indels

#### GATK HaplotypeCaller

GATK HaplotypeCaller is a robust germline variant caller that identifies SNPs and indels. It reconstructs haplotypes to ensure accurate variant calling, even in challenging genomic regions.

<details markdown="1">
<summary>Output files for GATK4 HaplotypeCaller:</summary>

  - `*_haplotypecaller.vcf.tbi`: index file for germline vcf.

</details>

### Structural Variant Calling

#### GRIDSS

GRIDSS is a structural variant caller that identifies genomic rearrangements such as deletions, duplications, insertions, inversions, and translocations. It is widely used in cancer genomics and other structural variation studies. By default, outputs from GRIDSS are not published.

#### GRIPSS

GRIPSS filters and refines structural variant calls from GRIDSS, separating somatic and germline variants. It improves the specificity and accuracy of results, making it particularly useful for analyzing complex rearrangements in cancer genomes.

<details markdown="1">
<summary>Output files for GRIPSS somatic and germline </summary>

- `{outdir}/svs/somatic/`
  - `*.gripss.filtered.somatic.vcf.gz`: filtered somatic structural variants vcf file
  - `*.gripss.filtered.somatic.vcf.gz.tbi`: filtered somatic structural variants vcf's index
  - `*.gripss.somatic.vcf.gz`: Unfiltered somatic structural variants vcf file
  - `*.gripss.somatic.vcf.gz.tbi`: Unfiltered somatic structural variants vcf's index file
- `{outdir}/svs/germline/`
  - `*.gripss.filtered.germline.vcf.gz`: filtered germline structural variants vcf file
  - `*.gripss.filtered.germline.vcf.gz.tbi`: filtered germline structural variants vcf's index
  - `*.gripss.germline.vcf.gz`: unfiltered germline structural variants vcf file
  - `*.gripss.germline.vcf.gz.tbi`: unfiltered germline structural variants vcf's index

</details>

### Copy Number Variant Calling

#### Jumble

Jumble is a tool for detecting copy number variants (CNVs) in sequencing data. It identifies regions of the genome with altered copy numbers, which are often associated with diseases like cancer.

<details markdown="1">
<summary>Output files for Jumble CNV calling: </summary>

- `{outdir}/cnv/`
  - `*_ann.cns`: Annotated copy number segments file with additional curation specific comments.
  - `*.cnr`: Copy number ratio file containing read depth ratios.
  - `*.cns`: Copy number segments file with called segments and their copy numbers.
  - `*_dnacopy.seg`: Segmentation file from DNAcopy algorithm.
  - `*.png`: Visualization plots of copy number profiles and segments.
  - `*_profile.bedgraph`: BedGraph file with copy number profile data for genome browser visualization.
  - `*_segments.bedgraph`: BedGraph file with segmented copy number regions for genome browser visualization.

</details>

### Tumor Purity/Ploidy Estimation

#### PureCN

PureCN estimates tumor purity and ploidy, providing critical information for interpreting cancer genomic data. It helps distinguish between somatic and germline variants in mixed samples. This local module accepts a pre-computed segmentation file from Jumble (rather than performing internal segmentation), and uses germline SNV calls from GATK Mutect2 for allele frequency estimation.

<details markdown="1">
<summary>Output files for PureCN:</summary>

- `{outdir}/purecn/`
  - `*.csv`: Tab-delimited summary file containing estimated tumor purity, ploidy, and quality flags for the sample.
  - `*.pdf`: Copy number profile plots showing coverage, segments, and allele frequencies across the genome.
  - `*_local_optima.pdf`: Plots of the purity/ploidy solution space, showing local optima considered during fitting.
  - `*_chromosomes.pdf`: Per-chromosome copy number plots. (optional)
  - `*_segmentation.pdf`: Segmentation diagnostic plots. (optional)
  - `*_genes.csv`: Gene-level copy number calls. (optional)
  - `*_amplification_pvalues.csv`: Amplification p-values per gene. (optional)
  - `*_variants.csv`: Variant-level output with allele-specific copy number assignments. (optional)
  - `*_loh.csv`: Loss-of-heterozygosity (LOH) calls per segment. (optional)
  - `*_dnacopy.seg`: Segmentation results in DNAcopy format. (optional)
  - `*.vcf.gz`: SNV calls with updated copy number and allele-specific information. (optional)
  - `*.log`: PureCN run log. (optional)

</details>

### Pharmacogenomics

#### typeDPYD

typeDPYD genotypes clinically relevant DPYD pharmacogenomic variants directly from a BAM file using pileup. It reports the DPYD diplotype and provides fluoropyrimidine dosing recommendations based on DPYD activity scores. Four SNPs are assessed: rs3918290 (*2A), rs55886062 (*13), rs67376798 (c.2846A>T), and rs56038477 (HapB3). Both hg19/GRCh37 and hg38/GRCh38 reference genome builds are supported and auto-detected from the BAM header.

<details markdown="1">
<summary>Output files for typeDPYD:</summary>

- `{outdir}/dpyd/`
  - `*.DPYD.csv`: Detailed evidence table with per-SNP pileup counts, allele fractions, copy calls, genotype assignment, and coverage summary for each sample.
  - `*.DPYD.json`: Structured JSON report containing the final DPYD genotype, clinical dosing recommendation text, and coverage summary, suitable for downstream reporting.

</details>

### Quality Control and Reporting

#### FastQC

FastQC is a quality control tool that provides metrics on sequencing reads, such as base quality scores, GC content, and adapter contamination. It helps assess the overall quality of sequencing data.

#### Picard - CollectHsMetrics, CollectMultipleMetrics

Picard's CollectHsMetrics and CollectMultipleMetrics calculate hybrid selection metrics and other quality control metrics. These tools are essential for evaluating the performance of sequencing experiments.

<details markdown="1">
<summary>Output files for picard tools:</summary>

- `{outdir}/qc/picard/`
  - `*.CollectHsMetrics.coverage_metrics`: Hybrid selection coverage metrics including on-target and off-target coverage statistics.
  - `*.CollectMultipleMetrics.alignment_summary_metrics`: Summary statistics of alignment quality and read mapping performance.
  - `*.CollectMultipleMetrics.base_distribution_by_cycle_metrics`: Distribution of bases across sequencing cycles for quality assessment.
  - `*.CollectMultipleMetrics.insert_size_metrics`: Fragment length distribution and insert size statistics for paired-end reads.
  - `*.CollectMultipleMetrics.quality_by_cycle_metrics`: Base quality scores stratified by sequencing cycle.
  - `*.CollectMultipleMetrics.quality_distribution_metrics`: Overall distribution of base quality scores across the dataset.

</details>

#### Samtools

Samtools is a suite of tools for manipulating and analyzing sequencing data in SAM/BAM formats. Samtools flagstats generates alignment statistics, providing metrics such as total reads, mapped reads, and properly paired reads. These statistics are essential for assessing alignment quality.

<details markdown="1">
<summary>Output files for Samtools:</summary>

- `{outdir}/qc/samtools/`
  - `*.flagstat`: Alignment statistics file containing read counts and mapping metrics.

</details>

#### GATK4 - CalculateContamination

GATK4's CalculateContamination estimates contamination levels in sequencing data. It is particularly useful for ensuring the accuracy of variant calling in mixed samples.

<details markdown="1">
<summary>Output files for contamination report:</summary>

- `{outdir}/qc/contamination/`
  - `*.contamination_table`: Table containing contamination estimates for each sample, including contamination fraction and confidence intervals.

</details>

#### MultiQC

[MultiQC](http://multiqc.info) is a visualization tool that generates a single HTML report summarising all samples in your project. Most of the pipeline QC results are visualised in the report and further statistics are available in the report data directory.

Results generated by MultiQC collate pipeline QC from supported tools e.g. FastQC. The pipeline has special steps which also allow the software versions to be reported in the MultiQC output for future traceability. For more information about how to use MultiQC reports, see <http://multiqc.info>.

<details markdown="1">
<summary>Output files</summary>

- `multiqc/`
  - `multiqc_report.html`: a standalone HTML file that can be viewed in your web browser.
  - `multiqc_data/`: directory containing parsed statistics from the different tools used in the pipeline.
  - `multiqc_plots/`: directory containing static images from the report in various formats.

</details>
